# DaggerHeart 自定义装备包用户指南

本指南面向会手写 JSON 的创作者。如果你希望把设定文稿交给 AI，让 AI 帮你生成装备包，请使用同目录下的 `AI-装备包生成提示词.md`。

装备包用于扩展角色卡可选择的武器和护甲。它和卡牌包是两种独立内容：卡牌包提供职业、种族、社群、子职业、领域和扩展卡牌；装备包提供可安装到角色表装备栏里的武器和护甲模板。

## 格式总览

装备包是一个 `.json` 文件。顶层必须包含：

```json
{
  "format": "daggerheart.equipment-pack.v1",
  "name": "装备包名称",
  "version": "1.0.0",
  "author": "作者",
  "description": "装备包描述",
  "equipment": {
    "weapons": [],
    "armor": []
  }
}
```

其中 `format` 必须固定为 `daggerheart.equipment-pack.v1`。系统会用它判断这是装备包，而不是卡牌包或其他 JSON。

`equipment.weapons` 和 `equipment.armor` 可以只写其中一类，但两者不能同时为空。一个装备包至少要包含一件武器或一件护甲。

## 顶层结构

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `format` | string | 是 | 固定为 `daggerheart.equipment-pack.v1` |
| `name` | string | 是 | 装备包名称，建议稳定不频繁改名 |
| `version` | string | 是 | 版本号，建议使用 `1.0.0` 这样的语义化版本 |
| `author` | string | 否 | 作者信息；缺失时会产生警告 |
| `description` | string | 否 | 装备包说明；缺失时会产生警告 |
| `equipment` | object | 是 | 装备内容容器 |
| `equipment.weapons` | array | 否 | 武器模板数组 |
| `equipment.armor` | array | 否 | 护甲模板数组 |

推荐始终填写 `author` 和 `description`。它们不一定阻止导入，但会帮助用户在内容包管理中识别来源。

## 最小可用装备包

下面是一个只包含一件武器的最小示例：

```json
{
  "format": "daggerheart.equipment-pack.v1",
  "name": "测试装备包",
  "version": "1.0.0",
  "author": "你的名字",
  "description": "用于测试装备包格式的最小示例。",
  "equipment": {
    "weapons": [
      {
        "id": "测试装备包-你的名字-weap-练习剑",
        "name": "练习剑",
        "tier": "T1",
        "weaponType": "primary",
        "trait": "agility",
        "damageType": "physical",
        "range": "melee",
        "burden": "oneHanded",
        "damage": "d8"
      }
    ],
    "armor": []
  }
}
```

下面是一个只包含一件护甲的最小示例：

```json
{
  "format": "daggerheart.equipment-pack.v1",
  "name": "测试护甲包",
  "version": "1.0.0",
  "author": "你的名字",
  "description": "用于测试护甲格式的最小示例。",
  "equipment": {
    "weapons": [],
    "armor": [
      {
        "id": "测试护甲包-你的名字-armo-练习甲",
        "name": "练习甲",
        "tier": "T1",
        "baseArmorMax": 4,
        "baseThresholds": {
          "minor": 7,
          "major": 15
        }
      }
    ]
  }
}
```

## 武器模板

武器模板写在 `equipment.weapons` 数组中。

```json
{
  "id": "装备包-作者-weap-影刃",
  "name": "影刃",
  "tier": "T1",
  "weaponType": "primary",
  "trait": "agility",
  "damageType": "physical",
  "range": "melee",
  "burden": "oneHanded",
  "damage": "d8",
  "featureName": "迅捷",
  "description": "攻击后可以描述一次灵巧的位移。"
}
```

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | string | 是 | 武器模板 ID，整个装备包内不能重复 |
| `name` | string | 是 | 武器名称 |
| `tier` | string | 是 | 阶级：`T1`、`T2`、`T3`、`T4` |
| `weaponType` | string | 是 | `primary` 主武器，`secondary` 副武器 |
| `trait` | string | 是 | 攻击属性 |
| `damageType` | string | 是 | `physical` 物理，`magic` 魔法 |
| `range` | string | 是 | 攻击范围 |
| `burden` | string | 是 | 负荷 |
| `damage` | string | 是 | 伤害骰或伤害表达式，如 `d8`、`d10+3` |
| `featureName` | string | 否 | 特性名称 |
| `description` | string | 否 | 特性描述 |
| `modifierContributions` | array | 否 | 角色数值修正 |

`featureName` 和 `description` 都可以省略，但建议至少写一个。否则导入时会产生“缺少模板描述”的警告。

## 护甲模板

护甲模板写在 `equipment.armor` 数组中。

```json
{
  "id": "装备包-作者-armo-影织甲",
  "name": "影织甲",
  "tier": "T1",
  "baseArmorMax": 4,
  "baseThresholds": {
    "minor": 7,
    "major": 15
  },
  "featureName": "轻盈",
  "description": "+1 闪避。"
}
```

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | string | 是 | 护甲模板 ID，整个装备包内不能重复 |
| `name` | string | 是 | 护甲名称 |
| `tier` | string | 是 | 阶级：`T1`、`T2`、`T3`、`T4` |
| `baseArmorMax` | integer | 是 | 基础护甲槽数量 |
| `baseThresholds.minor` | integer | 是 | 轻微伤害阈值 |
| `baseThresholds.major` | integer | 是 | 严重伤害阈值 |
| `featureName` | string | 否 | 特性名称 |
| `description` | string | 否 | 特性描述 |
| `modifierContributions` | array | 否 | 角色数值修正 |

护甲阈值有一个额外规则：`baseThresholds.major` 必须大于或等于 `baseThresholds.minor`。例如 `{ "minor": 7, "major": 15 }` 合法，`{ "minor": 15, "major": 7 }` 不合法。

## 枚举值与中文别名

推荐在正式装备包中使用英文标准值，因为它们和 schema 完全一致。导入器也支持下表中的中文别名，方便中文作者手写。

### 阶级 tier

| 值 | 说明 |
| :--- | :--- |
| `T1` | 一阶装备 |
| `T2` | 二阶装备 |
| `T3` | 三阶装备 |
| `T4` | 四阶装备 |

### 武器类型 weaponType

| 标准值 | 说明 |
| :--- | :--- |
| `primary` | 主武器 |
| `secondary` | 副武器 |

### 攻击属性 trait

| 标准值 | 中文别名 |
| :--- | :--- |
| `agility` | 敏捷 |
| `strength` | 力量 |
| `finesse` | 灵巧 |
| `instinct` | 本能 |
| `presence` | 风度 |
| `knowledge` | 知识 |

### 伤害类型 damageType

| 标准值 | 中文别名 |
| :--- | :--- |
| `physical` | 物理 |
| `magic` | 魔法 |

### 范围 range

| 标准值 | 中文别名 |
| :--- | :--- |
| `melee` | 近战 |
| `veryClose` | 邻近 |
| `close` | 近距离 |
| `far` | 远距离 |
| `veryFar` | 极远 |

### 负荷 burden

| 标准值 | 中文别名 |
| :--- | :--- |
| `oneHanded` | 单手 |
| `twoHanded` | 双手 |
| `offHand` | 副手 |

## 数值修正 modifierContributions

`modifierContributions` 用来描述装备带来的角色数值修正。它不会在装备包导入时直接改变角色；只有当玩家把这件装备选择到角色卡装备栏后，修正才会进入角色表的自动计算。

示例：一件护甲提供 `+1 闪避`。

```json
"modifierContributions": [
  {
    "id": "影织甲-mod-evasion",
    "definition": {
      "target": "evasion",
      "kind": "modifier"
    },
    "editable": {
      "label": "轻盈",
      "value": 1
    }
  }
]
```

示例：一件重甲提供 `+1 护甲槽`，同时 `-1 闪避`。

```json
"modifierContributions": [
  {
    "id": "星铁重甲-mod-armorMax",
    "definition": {
      "target": "armorMax",
      "kind": "modifier"
    },
    "editable": {
      "label": "沉稳",
      "value": 1
    }
  },
  {
    "id": "星铁重甲-mod-evasion",
    "definition": {
      "target": "evasion",
      "kind": "modifier"
    },
    "editable": {
      "label": "沉稳",
      "value": -1
    }
  }
]
```

### 修正项字段

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | string | 是 | 修正项 ID，同一件装备内不能重复 |
| `definition.target` | string | 是 | 修正目标 |
| `definition.kind` | string | 是 | 固定为 `modifier` |
| `editable.label` | string | 是 | 在角色卡上显示的修正来源名称 |
| `editable.value` | number | 是 | 修正数值，可以是正数、负数或 0 |

### 可用修正目标

| target | 说明 |
| :--- | :--- |
| `evasion` | 闪避 |
| `armorMax` | 护甲槽上限 |
| `minorThreshold` | 轻微阈值 |
| `majorThreshold` | 严重阈值 |
| `hpMax` | 生命上限 |
| `stressMax` | 压力上限 |
| `proficiency` | 熟练 |
| `agility.value` | 敏捷值 |
| `strength.value` | 力量值 |
| `finesse.value` | 灵巧值 |
| `instinct.value` | 本能值 |
| `presence.value` | 风度值 |
| `knowledge.value` | 知识值 |

## ID 与版本规范

### 装备模板 ID

`id` 是装备模板的稳定身份。系统会用它识别这件装备。建议使用可读、稳定、不容易和别人冲突的格式：

```text
包名-作者-weap-装备名
包名-作者-armo-装备名
```

例如：

```text
中文装备包-测试作者-weap-影刃
中文装备包-测试作者-armo-影织甲
```

注意：

- 同一个装备包内，武器和护甲的 `id` 不能重复。
- `id` 不能包含 `/`、反斜杠、控制字符。
- 如果 `id` 和内置装备或已安装装备包冲突，导入会失败。
- 发布新版本时，除非你希望系统把它视为另一件装备，否则不要随意修改已有装备的 `id`。

### 修正项 ID

修正项 `id` 只要求在同一件装备的 `modifierContributions` 内不重复。为了排查问题方便，仍建议写得可读：

```text
装备ID-mod-evasion
装备ID-mod-armorMax
```

### 版本号

`version` 必须是语义化版本字符串，例如：

```text
1.0.0
1.1.0
2.0.0-beta.1
```

常见的 `v1`、`第一版`、`2026版` 都不是合法版本号。

## 验证规则与常见错误

导入装备包时，系统会先做结构校验，再做语义校验。编辑器能打开 JSON 不代表它一定可以正式导入；请以“验证”或正式导入结果为准。

### 会阻止导入的错误

| 错误 | 说明 | 修正方式 |
| :--- | :--- | :--- |
| `format` 错误 | 不是 `daggerheart.equipment-pack.v1` | 改成固定格式标识 |
| 缺少必填字段 | 如武器缺少 `damage` | 补齐字段 |
| 多余字段 | schema 不允许未知字段 | 删除拼错或未支持的字段 |
| 枚举值错误 | 如 `trait` 写成 `智力` | 使用合法枚举或支持的中文别名 |
| 数字类型错误 | 如 `"baseArmorMax": "4"` | 改成 `"baseArmorMax": 4` |
| 装备包为空 | 武器和护甲都为空 | 至少添加一件武器或护甲 |
| ID 重复 | 多件装备使用同一 `id` | 修改其中一个 ID |
| ID 冲突 | 和内置或已安装装备冲突 | 使用更独特的 ID |
| 护甲阈值顺序错误 | `major < minor` | 调整为 `major >= minor` |
| 版本号错误 | 不是语义化版本 | 使用 `1.0.0` 这样的格式 |

### 可能出现的警告

| 警告 | 说明 |
| :--- | :--- |
| 缺少作者 | `author` 没填 |
| 缺少描述 | `description` 没填 |
| 装备缺少描述 | `featureName` 和 `description` 都为空 |
| 描述较长 | 描述文本过长，可能影响阅读 |

警告不一定阻止导入，但建议在发布前处理。

## Schema

装备包指南目录中提供了一份可下载的 JSON Schema：

```text
/自定义装备包指南和示例/equipment-pack.v1.schema.json
```

项目内的规范来源文件位于：

```text
/schemas/equipment-pack.v1.schema.json
```

如果你使用 VS Code、JetBrains IDE、在线 JSON 校验器或 AI 工具，可以把 schema 作为最终校验规则。本文档解释创作意图和常见写法，schema 是机器校验的准绳。

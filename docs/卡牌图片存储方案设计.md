# 卡牌图片存储方案设计文档

## 一、背景与需求分析

### 1.1 现状分析

当前卡牌系统完全基于 localStorage 存储，存在以下限制：

| 方面 | 现状 | 问题 |
|------|------|------|
| **存储容量** | localStorage 限制 5-10MB | 无法存储大量图片数据 |
| **数据格式** | 仅支持字符串 | 图片需 base64 编码，体积增加 33% |
| **性能影响** | 同步 API | 大数据读写阻塞主线程 |
| **用户体验** | 图片通过 URL 引用 | 需要额外部署图片资源 |

### 1.2 需求定义

#### 核心需求
1. **支持图片打包**：用户可以导入包含图片的 ZIP 格式卡包
2. **本地化存储**：图片数据完全存储在用户本地
3. **向后兼容**：不破坏现有功能和数据
4. **高性能**：图片加载不影响界面响应

#### 约束条件
1. **最小化改动**：保持现有代码架构，降低重构风险
2. **渐进式升级**：支持新旧系统并存
3. **数据完整性**：确保批次级整存整取，避免孤儿数据
4. **浏览器兼容**：支持现代浏览器（Chrome 90+, Firefox 88+, Safari 14+）

### 1.3 设计目标

1. **混合存储**：元数据保留在 localStorage，图片存储在 IndexedDB
2. **透明升级**：对上层业务逻辑透明，API 保持兼容
3. **智能缓存**：实现高效的图片缓存和内存管理
4. **数据安全**：提供完整的数据一致性保证

## 二、技术方案选型

### 2.1 存储方案对比

| 方案 | 优势 | 劣势 | 选择理由 |
|------|------|------|----------|
| **纯 localStorage** | 简单、同步访问 | 容量限制、无法存储二进制 | ❌ 不满足需求 |
| **纯 IndexedDB** | 容量大、支持二进制 | 需要重构所有代码、异步访问 | ❌ 改动太大 |
| **混合方案** ⭐ | 改动最小、兼容性好 | 需要管理两个存储系统 | ✅ **最佳选择** |

### 2.2 IndexedDB 封装库选型

### 2.3 文件格式设计

采用 ZIP 格式作为卡包标准格式：

```
card-pack.zip
├── metadata.json          # 卡包元数据
├── manifest.json          # 文件清单（可选）
└── cards/
    ├── profession/
    │   ├── card1.json    # 卡牌数据
    │   └── card1.webp    # 卡牌图片
    ├── ancestry/
    │   ├── card2.json
    │   └── card2.webp
    └── ...
```

**格式选择理由**：
- WebP：最佳压缩率，浏览器原生支持
- JSON：结构化数据，易于解析
- ZIP：标准压缩格式，工具支持广泛

### 4.3 批次管理策略

```typescript
class BatchImageManager {
  private db: Dexie;
  
  // 原子性导入：使用事务确保要么全部成功，要么全部失败
  async importBatch(batchId: string, cards: CardData[], images: Map<string, Blob>) {
    return await this.db.transaction('rw', 
      this.db.images, 
      this.db.batchRegistry, 
      async () => {
        // 1. 注册批次
        const imageIds = [];
        const cardIds = cards.map(c => c.id);
        
        await this.db.batchRegistry.put({
          batchId,
          cardIds,
          imageIds: [],
          totalSize: 0,
          importTime: Date.now()
        });
        
        // 2. 批量存储图片
        let totalSize = 0;
        for (const [cardId, blob] of images) {
          const imageId = `${batchId}:${cardId}`;
          imageIds.push(imageId);
          totalSize += blob.size;
          
          await this.db.images.put({
            id: imageId,
            batchId,
            cardId,
            blob,
            mimeType: blob.type,
            size: blob.size,
            createdAt: Date.now()
          });
        }
        
        // 3. 更新批次信息
        await this.db.batchRegistry.update(batchId, {
          imageIds,
          totalSize
        });
        
        // 4. 更新 localStorage 中的卡牌数据
        this.updateCardReferences(cards, batchId);
        
        return { success: true, imageCount: imageIds.length };
      }
    );
  }
  
  // 原子性删除
  async deleteBatch(batchId: string) {
    return await this.db.transaction('rw',
      this.db.images,
      this.db.batchRegistry,
      async () => {
        // 1. 获取批次信息
        const registry = await this.db.batchRegistry.get(batchId);
        if (!registry) return false;
        
        // 2. 删除所有图片
        await this.db.images.bulkDelete(registry.imageIds);
        
        // 3. 删除批次注册
        await this.db.batchRegistry.delete(batchId);
        
        // 4. 清理缓存
        this.clearBatchCache(batchId);
        
        return true;
      }
    );
  }
}
```

### 4.4 图片缓存机制

```typescript
class ImageCacheManager {
  private cache = new Map<string, CacheEntry>();
  private maxMemory = 50 * 1024 * 1024; // 50MB
  private currentMemory = 0;
  
  interface CacheEntry {
    url: string;          // Object URL
    blob: Blob;          // 原始数据
    size: number;        // 大小
    refCount: number;    // 引用计数
    lastAccess: number;  // LRU 时间戳
    priority: number;    // 优先级
  }
  
  // 智能缓存策略
  async getOrCreate(imageId: string): Promise<string> {
    // 1. 检查缓存
    if (this.cache.has(imageId)) {
      const entry = this.cache.get(imageId);
      entry.refCount++;
      entry.lastAccess = Date.now();
      return entry.url;
    }
    
    // 2. 从 IndexedDB 加载
    const image = await this.db.images.get(imageId);
    if (!image) return null;
    
    // 3. 检查内存限制
    if (this.currentMemory + image.size > this.maxMemory) {
      this.evictLRU(image.size);
    }
    
    // 4. 创建缓存项
    const url = URL.createObjectURL(image.blob);
    const entry: CacheEntry = {
      url,
      blob: image.blob,
      size: image.size,
      refCount: 1,
      lastAccess: Date.now(),
      priority: this.calculatePriority(image)
    };
    
    this.cache.set(imageId, entry);
    this.currentMemory += image.size;
    
    return url;
  }
  
  // LRU 驱逐策略
  private evictLRU(requiredSpace: number) {
    const entries = Array.from(this.cache.entries())
      .filter(([_, entry]) => entry.refCount === 0)
      .sort((a, b) => {
        // 优先级低且最久未使用的先驱逐
        const scoreA = a[1].priority * 1000 + (Date.now() - a[1].lastAccess);
        const scoreB = b[1].priority * 1000 + (Date.now() - b[1].lastAccess);
        return scoreB - scoreA;
      });
    
    let freedSpace = 0;
    for (const [id, entry] of entries) {
      URL.revokeObjectURL(entry.url);
      this.cache.delete(id);
      freedSpace += entry.size;
      this.currentMemory -= entry.size;
      
      if (freedSpace >= requiredSpace) break;
    }
  }
  
  // 预加载策略
  async preloadBatch(batchId: string, priority: 'high' | 'low' = 'low') {
    if (priority === 'high') {
      // 立即加载
      const images = await this.db.images
        .where('batchId')
        .equals(batchId)
        .toArray();
      
      for (const image of images) {
        await this.getOrCreate(image.id);
      }
    } else {
      // 空闲时加载
      requestIdleCallback(async () => {
        const images = await this.db.images
          .where('batchId')
          .equals(batchId)
          .limit(10)  // 分批加载
          .toArray();
        
        for (const image of images) {
          await this.getOrCreate(image.id);
        }
      });
    }
  }
  
  // 释放引用
  release(imageId: string) {
    const entry = this.cache.get(imageId);
    if (entry) {
      entry.refCount--;
      
      // 延迟清理，避免频繁创建/销毁
      if (entry.refCount === 0) {
        setTimeout(() => {
          if (entry.refCount === 0) {
            URL.revokeObjectURL(entry.url);
            this.cache.delete(imageId);
            this.currentMemory -= entry.size;
          }
        }, 30000); // 30秒后清理
      }
    }
  }
}
```

### 5.2 数据一致性检查

```typescript
class IntegrityChecker {
  // 定期检查数据一致性
  async checkIntegrity(): Promise<IntegrityReport> {
    const report = {
      healthy: true,
      issues: [],
      orphanedImages: [],
      missingImages: [],
      suggestions: []
    };
    
    // 1. 检查 localStorage 和 IndexedDB 的对应关系
    const localBatches = this.getLocalStorageBatches();
    const indexedBatches = await this.getIndexedDBBatches();
    
    // 2. 找出孤儿图片（IndexedDB 有但 localStorage 没有）
    for (const batch of indexedBatches) {
      if (!localBatches.has(batch.batchId)) {
        report.orphanedImages.push(...batch.imageIds);
        report.issues.push(`Orphaned batch: ${batch.batchId}`);
      }
    }
    
    // 3. 找出缺失图片（localStorage 标记有但 IndexedDB 没有）
    for (const [batchId, cards] of localBatches) {
      for (const card of cards) {
        if (card.imageStorage?.hasImage) {
          const imageId = card.imageStorage.imageId;
          const exists = await this.db.images.get(imageId);
          if (!exists) {
            report.missingImages.push(imageId);
            report.issues.push(`Missing image: ${imageId}`);
          }
        }
      }
    }
    
    // 4. 生成修复建议
    if (report.orphanedImages.length > 0) {
      report.suggestions.push('Run cleanOrphans() to remove orphaned images');
    }
    if (report.missingImages.length > 0) {
      report.suggestions.push('Re-import affected card packs');
    }
    
    report.healthy = report.issues.length === 0;
    return report;
  }
  
  // 清理孤儿数据
  async cleanOrphans(): Promise<CleanupResult> {
    const report = await this.checkIntegrity();
    let cleaned = 0;
    let freedSpace = 0;
    
    for (const imageId of report.orphanedImages) {
      const image = await this.db.images.get(imageId);
      if (image) {
        freedSpace += image.size;
        await this.db.images.delete(imageId);
        cleaned++;
      }
    }
    
    return { cleaned, freedSpace };
  }
}
```

```typescript
class ErrorRecovery {

  async rollbackImport(batchId: string) {
    try {
      // 1. 删除 IndexedDB 中的部分数据
      await this.imageService.deleteBatchImages(batchId);

      return { success: true };

      console.error('Rollback failed:', error);
      return { success: false, error };
    }
  }
  
  // 自动修复
  async autoRepair() {
    const report = await this.integrityChecker.checkIntegrity();
    
    if (!report.healthy) {
      // 1. 清理孤儿数据
      if (report.orphanedImages.length > 0) {
        await this.integrityChecker.cleanOrphans();
      }
      
      // 2. 修复缺失的图片引用
      for (const cardId of report.missingImages) {
        await this.fixMissingImageReference(cardId);
      }
      
      // 3. 重建索引
      await this.rebuildIndexes();
    }
  }
```

## 七、性能优化

### 7.1 缓存策略

#### 三级缓存架构
```typescript
class ThreeLevelCache {
  // L1: 内存缓存（Object URLs）
  private memoryCache = new Map<string, string>();
  
  // L2: IndexedDB 缓存
  private indexedDB: Dexie;
  
  // L3: 原始文件（ZIP 导入时保留）
  private sourceFiles = new Map<string, File>();
  
  async getImage(cardId: string): Promise<string> {
    // 1. 检查 L1 缓存
    if (this.memoryCache.has(cardId)) {
      return this.memoryCache.get(cardId);
    }
    
    // 2. 检查 L2 缓存
    const cached = await this.indexedDB.images.get(cardId);
    if (cached) {
      const url = URL.createObjectURL(cached.blob);
      this.memoryCache.set(cardId, url);
      return url;
    }
    
    // 3. 从源文件恢复（如果有）
    if (this.sourceFiles.has(cardId)) {
      // 重新导入
      await this.reimportFromSource(cardId);
```

```typescript
class LazyImageLoader {
  private loadQueue: LoadTask[] = [];
  private isLoading = false;
  
  // 基于优先级的加载队列
  async queueLoad(cardId: string, priority: Priority = 'normal') {
    const task: LoadTask = {
      cardId,
      priority,
      timestamp: Date.now()
    };
    
    // 插入到合适的位置
    const index = this.findInsertIndex(priority);
    this.loadQueue.splice(index, 0, task);
    
    // 启动加载
    if (!this.isLoading) {
      this.processQueue();
    }
  }
  
  // 可见性驱动的加载
  observeVisibility(element: HTMLElement, cardId: string) {
    const observer = new IntersectionObserver(
      (entries) => {
        if (entries[0].isIntersecting) {
          // 元素可见，高优先级加载
          this.queueLoad(cardId, 'high');
          observer.disconnect();
        }
      },
      { rootMargin: '50px' }  // 提前 50px 开始加载
    );
    
    observer.observe(element);
  }
  
  // 批量处理队列
  private async processQueue() {
    this.isLoading = true;
    
    while (this.loadQueue.length > 0) {
      // 批量处理高优先级任务
      const batch = this.loadQueue
        .filter(t => t.priority === 'high')
        .slice(0, 5);
      
      if (batch.length === 0) {
        // 处理普通优先级
        batch.push(...this.loadQueue.slice(0, 3));
      }
      
      await Promise.all(
        batch.map(task => this.loadImage(task.cardId))
      );
      
      // 从队列中移除
      batch.forEach(task => {
        const index = this.loadQueue.indexOf(task);
        if (index > -1) {
          this.loadQueue.splice(index, 1);
        }
      });
      
      // 让出控制权
      await new Promise(resolve => setTimeout(resolve, 10));
    }
    
    this.isLoading = false;
  }
}
```

```typescript
class MemoryManager {
  private memoryLimit = 50 * 1024 * 1024; // 50MB
  private warningThreshold = 0.8;         // 80% 警告
  private criticalThreshold = 0.95;       // 95% 关键
  
  // 监控内存使用
  startMonitoring() {
    setInterval(() => {
      const usage = this.getMemoryUsage();
      
      if (usage.ratio > this.criticalThreshold) {
        this.emergencyCleanup();
      } else if (usage.ratio > this.warningThreshold) {
        this.preventiveCleanup();
      }
    }, 5000);
  }
  
  // 预防性清理
  private preventiveCleanup() {
    // 清理未使用的缓存
    this.cacheManager.evictUnused();
    
    // 降低预加载积极性
    this.preloadManager.reducePriority();
  }
  
  // 紧急清理
  private emergencyCleanup() {
    // 清理所有非关键缓存
    this.cacheManager.clearNonEssential();
    
    // 停止所有预加载
    this.preloadManager.stopAll();
    
    // 触发垃圾回收（如果可能）
    if (window.gc) {
      window.gc();
    }
  }
  
  // 获取内存使用情况
  private getMemoryUsage() {
    const used = this.cacheManager.getCurrentSize();
    return {
      used,
      limit: this.memoryLimit,
      ratio: used / this.memoryLimit
    };
  }
}
```

```typescript
class MigrationService {
  // 检测并执行迁移
  async migrate() {
    const migrationStatus = this.getMigrationStatus();
    
    switch (migrationStatus) {
      case 'not_needed':
        console.log('No migration needed');
        break;
        
      case 'partial':
        await this.continuePartialMigration();
        break;
        
      case 'required':
        await this.startFullMigration();
        break;
    }
  }
  
  // 渐进式迁移
  async startFullMigration() {
    const batches = this.store.getAllBatches();
    const total = batches.length;
    let completed = 0;
    
    for (const batch of batches) {
      try {
        // 1. 迁移批次
        await this.migrateBatch(batch);
        
        // 2. 更新进度
        completed++;
        this.onProgress?.(completed / total * 100);
        
        // 3. 保存检查点
        this.saveMigrationCheckpoint(batch.id);
        
      } catch (error) {
        console.error(`Failed to migrate batch ${batch.id}:`, error);
        
        // 继续迁移其他批次
        this.logMigrationError(batch.id, error);
      }
    }
    
    // 标记迁移完成
    this.markMigrationComplete();
  }
  
  // 迁移单个批次
  private async migrateBatch(batch: BatchInfo) {
    const cards = this.store.getCardsByBatch(batch.id);
    const imagesToDownload = [];
    
    // 收集需要下载的图片
    for (const card of cards) {
      if (card.imageUrl && !card.imageStorage?.hasImage) {
        imagesToDownload.push({
          cardId: card.id,
          url: card.imageUrl
        });
      }
    }
    
    if (imagesToDownload.length === 0) {
      return; // 无需迁移
    }
    
    // 批量下载和存储
    const images = new Map();
    for (const item of imagesToDownload) {
      try {
        const blob = await this.downloadImage(item.url);
        images.set(item.cardId, blob);
      } catch (error) {
        console.warn(`Failed to download image for ${item.cardId}`);
      }
    }
    
    // 批量存储到 IndexedDB
    if (images.size > 0) {
      await this.imageService.importBatchImages(batch.id, images);
      
      // 更新卡牌引用
      this.updateCardReferences(cards, batch.id);
    }
  }
}
```

```typescript
class DataValidator {
  // 验证 ZIP 文件
  async validateZipFile(file: File): Promise<ValidationResult> {
    // 1. 文件大小限制
    if (file.size > 100 * 1024 * 1024) {
      return { valid: false, error: 'File too large (max 100MB)' };
    }
    
    // 2. 文件类型检查
    if (!file.name.endsWith('.zip')) {
      return { valid: false, error: 'Invalid file type' };
    }
    
    // 3. 内容验证
    try {
      const zip = await JSZip.loadAsync(file);
      
      // 检查必需文件
      if (!zip.file('metadata.json')) {
        return { valid: false, error: 'Missing metadata.json' };
      }
      
      // 验证图片格式
      const images = zip.filter((path) => 
        path.match(/\.(webp|png|jpg|jpeg)$/i)
      );
      
      for (const imagePath of images) {
        const imageFile = zip.file(imagePath);
        const blob = await imageFile.async('blob');
        
        // 验证 MIME 类型
        if (!blob.type.startsWith('image/')) {
          return { 
            valid: false, 
            error: `Invalid image type: ${imagePath}` 
          };
        }
        
        // 大小限制（单个图片 5MB）
        if (blob.size > 5 * 1024 * 1024) {
          return { 
            valid: false, 
            error: `Image too large: ${imagePath}` 
          };
        }
      }
      
      return { valid: true };
    } catch (error) {
      return { valid: false, error: 'Failed to parse ZIP file' };
    }
  }
  
  // 验证卡牌数据
  validateCardData(card: any): boolean {
    // 必需字段
    if (!card.id || !card.name || !card.type) {
      return false;
    }
    
    // ID 格式
    if (!/^[a-zA-Z0-9_-]+$/.test(card.id)) {
      return false;
    }
    
    // 类型验证
```

```typescript
      });

      }
    }
```

## 十一、总结

### 11.1 方案优势

1. **最小化改动**：保持现有 localStorage 架构，仅添加 IndexedDB 图片存储
2. **向后兼容**：支持新旧系统并存，平滑过渡
3. **高性能**：多级缓存、智能预加载、LRU 淘汰
4. **数据安全**：事务保证、完整性检查、错误恢复
5. **用户友好**：支持 ZIP 导入导出，进度显示

### 11.2 技术栈

- **存储**：localStorage（元数据）+ IndexedDB（图片）
- **库**：Dexie.js（IndexedDB）+ JSZip（文件处理）
- **模式**：混合存储、批次管理、缓存优化

### 11.3 关键指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 图片加载时间 | < 100ms | 缓存命中时 < 10ms |
| 内存占用 | < 50MB | 自动清理保持在限制内 |
| 导入速度 | > 10MB/s | 批量处理优化 |

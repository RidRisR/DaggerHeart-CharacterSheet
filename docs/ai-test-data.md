## 测试建议

1. **从小到大**: 先测试小文本，确保基本功能正常
2. **逐步增加复杂度**: 再测试中等文本，验证分块和合并
3. **边界测试**: 测试边界情况，确保健壮性
4. **错误测试**: 测试错误输入，验证错误处理

**推荐测试顺序**:
1. 小文本测试 (必测)
2. 中等文本测试 (必测)
3. 边界情况测试1-3
4. 错误测试1-2
5. 大文本测试 (可选)

# 卡牌包加载和读取流程

## 系统架构概述

卡牌系统使用 Zustand store 进行状态管理，所有卡牌数据存储在 localStorage 中。系统支持内置卡牌包和自定义卡牌包，使用统一的接口进行管理。

### 核心文件
- `card/index.ts` - 系统入口，导出 unified 实现
- `card/index-unified.ts` - 统一卡牌系统的 API 接口
- `card/stores/unified-card-store.ts` - Zustand store 定义
- `card/stores/store-actions.ts` - 所有业务逻辑和操作
- `app/card-manager/page.tsx` - 卡牌管理界面

### 数据存储结构
```javascript
localStorage: {
  // 索引文件 - 记录所有批次的元信息
  'daggerheart_custom_cards_index': {
    batches: {
      [batchId]: {
        id, name, fileName, importTime,
        cardCount, cardTypes, size,
        isSystemBatch, disabled
      }
    },
    totalCards, totalBatches, lastUpdate
  },
  
  // 批次数据 - 每个批次的详细数据
  'daggerheart_batch_[batchId]': {
    metadata: { id, name, fileName, ... },
    cards: [...],  // 卡牌数组
    customFieldDefinitions: {...},
    variantTypes: {...}
  }
}
```

## 二、加载现有卡牌包流程

### 2.1 系统初始化时加载

系统在初始化时（`initializeSystem()`）会自动加载所有卡牌包：

```typescript
initializeSystem() {
  // 1. 检查旧版数据迁移
  _migrateLegacyData();
  
  // 2. 统一加载所有卡牌
  _loadAllCards() {
    // 2.1 先加载内置卡牌（确保显示顺序）
    _seedBuiltinCards();
    
    // 2.2 再加载自定义卡牌
    _loadCustomCardsFromStorage();
  }
  
  // 3. 重新计算聚合数据
  _recomputeAggregations();
  
  // 4. 重建类型索引
  _rebuildCardsByType();
  
  // 5. 预处理卡牌图片
  _preprocessCardImages();
  
  // 6. 同步到 localStorage
  _syncToLocalStorage();
}
```

### 2.2 内置卡牌包特殊处理

#### 内置卡牌包的特点
- **固定 ID**: 使用常量 `BUILTIN_BATCH_ID = "SYSTEM_BUILTIN_CARDS"`
- **数据来源**: 从 `data/cards/builtin-base.json` 导入
- **系统批次标记**: `isSystemBatch: true`
- **不可删除**: UI 中删除按钮对系统批次禁用
- **可以禁用**: 支持启用/禁用状态切换
- **优先加载**: 总是在自定义卡牌之前加载，确保显示顺序

#### 内置卡牌包数据结构
```json
{
  "name": "系统内置卡牌包",
  "version": "V20250904",
  "description": "系统内置卡牌包,数据来自官方SRD",
  "author": "RidRisR,PolearmMaster,末楔,里予,一得",
  "customFieldDefinitions": {
    "variantTypes": { /* 变体类型定义 */ },
    "professions": [ /* 职业列表 */ ],
    "ancestries": [ /* 种族列表 */ ],
    "communities": [ /* 社群列表 */ ]
  },
  /* 各类型卡牌数据 */
}
```

#### 加载内置卡牌包流程

```typescript
_seedBuiltinCards() {
  // 1. 检查并恢复保存的禁用状态
  let savedDisabledStatus = false;
  const indexStr = localStorage.getItem('daggerheart_custom_cards_index');
  if (indexStr) {
    const index = JSON.parse(indexStr);
    // 恢复之前保存的内置包禁用状态
    savedDisabledStatus = index.batches?.[BUILTIN_BATCH_ID]?.disabled || false;
  }
  
  // 2. 移除已存在的内置批次（如果有）
  if (state.batches.has(BUILTIN_BATCH_ID)) {
    get().removeBatch(BUILTIN_BATCH_ID);
  }
  
  // 3. 导入内置卡牌 JSON
  const builtinCardPack = await import('builtin-base.json');
  
  // 4. 导入内置卡牌
  _importBuiltinCards(builtinCardPack, savedDisabledStatus) {
    // 创建内置批次
    const batchInfo = {
      id: BUILTIN_BATCH_ID,
      name: pack.name,
      isSystemBatch: true,  // 关键标记
      disabled: savedDisabledStatus, // 恢复禁用状态
      ...
    };
    
    // 添加卡牌并标记为内置
    cards.forEach(card => {
      newCards.set(card.id, {
        ...card,
        batchId: BUILTIN_BATCH_ID,
        source: CardSource.BUILTIN  // 标记为内置来源
      });
    });
  }
}
```

#### 内置卡牌包的持久化
- **禁用状态保存**: 内置包的禁用状态会保存在索引中
- **卡牌数据不保存**: 内置卡牌数据每次从 JSON 文件重新加载
- **加载顺序保证**: 自定义卡牌加载时会跳过 `BUILTIN_BATCH_ID`

### 2.3 加载自定义卡牌包

```typescript
_loadCustomCardsFromStorage() {
  // 1. 读取索引
  const index = JSON.parse(localStorage.getItem(STORAGE_KEYS.INDEX));
  
  // 2. 检测孤立的批次键（在 localStorage 但不在索引中）
  const orphanedKeys = detectOrphanedKeys();
  
  // 3. 遍历索引中的批次
  for (const batchId of Object.keys(index.batches)) {
    // 跳过内置批次（已经加载）
    if (batchId === BUILTIN_BATCH_ID) continue;
    
    // 4. 读取批次数据
    const batchData = JSON.parse(
      localStorage.getItem(`${STORAGE_KEYS.BATCH_PREFIX}${batchId}`)
    );
    
    // 5. 创建批次信息
    const batch = {
      id: batchData.metadata.id,
      name: batchInfo.name,
      disabled: batchInfo.disabled,
      cardIds: batchData.cards.map(c => c.id),
      ...
    };
    
    // 6. 添加卡牌到全局 Map
    batchData.cards.forEach(card => {
      cards.set(card.id, {
        ...card,
        batchId,
        source: CardSource.CUSTOM
      });
    });
  }
}
```

## 三、卡牌查询接口

### 3.1 获取所有卡牌
```typescript
loadAllCards() {
  // 从 cards Map 获取所有卡牌
  const allCards = Array.from(state.cards.values());
  
  // 过滤掉禁用批次的卡牌
  return allCards.filter(card => {
    if (card.batchId) {
      const batch = state.batches.get(card.batchId);
      return !batch?.disabled;
    }
    return true;
  });
}
```

### 3.2 按类型获取卡牌
```typescript
loadCardsByType(type: CardType) {
  // 使用预构建的类型索引 Map（O(1) 查找）
  const typeCardIds = state.cardsByType.get(type) || [];
  
  // 通过 ID 获取卡牌并过滤
  return typeCardIds
    .map(cardId => state.cards.get(cardId))
    .filter(card => {
      // 过滤禁用批次的卡牌
      if (card.batchId) {
        const batch = state.batches.get(card.batchId);
        return !batch?.disabled;
      }
      return true;
    });
}
```

### 3.3 按 ID 获取单个卡牌
```typescript
getCardById(cardId: string) {
  // O(1) 查找
  const card = state.cards.get(cardId);
  
  // 检查批次是否禁用
  if (card?.batchId) {
    const batch = state.batches.get(card.batchId);
    if (batch?.disabled) return null;
  }
  
  return card;
}
```

### 3.4 按批次获取卡牌
```typescript
getCardsByBatchId(batchId: string) {
  return state.cards.values()
    .filter(card => card.batchId === batchId);
}
```

## 四、批次管理功能

### 4.1 切换批次启用/禁用状态

#### 基本流程
```typescript
toggleBatchDisabled(batchId: string) {
  const batch = state.batches.get(batchId);
  if (!batch) return false;
  
  // 切换禁用状态
  const newDisabled = !(batch.disabled || false);
  
  // 更新批次信息
  const updatedBatch = {
    ...batch,
    disabled: newDisabled
  };
  
  // 更新批次 Map
  newBatches.set(batchId, updatedBatch);
  
  // 更新索引中的禁用状态
  newIndex.batches[batchId].disabled = newDisabled;
  
  // 同步到 localStorage
  _syncToLocalStorage();
  
  // 标记缓存失效
  set({ cacheValid: false });
  
  return true;
}
```

#### 启用/禁用的影响
1. **卡牌查询**: 禁用批次的卡牌不会出现在查询结果中
   ```typescript
   loadAllCards() {
     return allCards.filter(card => {
       if (card.batchId) {
         const batch = state.batches.get(card.batchId);
         return !batch?.disabled;  // 过滤禁用批次
       }
       return true;
     });
   }
   ```

2. **聚合数据**: 禁用批次的自定义字段和变体类型不参与聚合
   ```typescript
   _recomputeAggregations() {
     for (const batch of state.batches.values()) {
       if (batch.disabled) continue;  // 跳过禁用批次
       // 聚合自定义字段和变体类型...
     }
   }
   ```

3. **UI 显示**: 禁用批次显示特殊标记
   ```jsx
   <Badge variant={batch.disabled ? "destructive" : "default"}>
     {batch.disabled ? "未启用" : "已启用"}
   </Badge>
   ```

#### 内置卡包的特殊处理
- **支持禁用**: 内置卡包也可以被禁用
- **状态持久化**: 内置卡包的禁用状态会保存并在下次加载时恢复
- **不可删除**: 即使禁用，内置卡包也不能被删除

### 4.2 删除批次
```typescript
removeBatch(batchId: string) {
  // 1. 删除批次
  batches.delete(batchId);
  
  // 2. 删除该批次的所有卡牌
  batch.cardIds.forEach(cardId => {
    cards.delete(cardId);
  });
  
  // 3. 更新索引
  delete index.batches[batchId];
  
  // 4. 从 localStorage 删除
  localStorage.removeItem(`${STORAGE_KEYS.BATCH_PREFIX}${batchId}`);
  
  // 5. 重建类型索引
  _rebuildCardsByType();
}
```

### 4.3 清空所有自定义卡牌
```typescript
clearAllCustomCards() {
  // 保留内置批次，删除所有自定义批次
  for (const [batchId] of batches) {
    if (batchId === BUILTIN_BATCH_ID) continue;  // 跳过内置批次
    
    // 从 localStorage 删除批次数据
    localStorage.removeItem(`${STORAGE_KEYS.BATCH_PREFIX}${batchId}`);
    batches.delete(batchId);
  }
  
  // 只保留内置卡牌
  const builtinBatch = state.batches.get(BUILTIN_BATCH_ID);
  if (builtinBatch) {
    newBatches.set(BUILTIN_BATCH_ID, builtinBatch);
    // 通过 cardIds 保留内置卡牌
    builtinBatch.cardIds.forEach(cardId => {
      const card = state.cards.get(cardId);
      if (card) {
        newCards.set(cardId, card);
      }
    });
  }
  
  // 更新索引，只保留内置批次
  const newIndex = {
    batches: builtinBatch ? {
      [BUILTIN_BATCH_ID]: { /* 内置批次信息 */ }
    } : {},
    totalCards: newCards.size,
    totalBatches: newBatches.size,
    lastUpdate: new Date().toISOString()
  };
}
```

## 五、性能优化策略

### 5.1 类型索引 Map
- 预构建 `cardsByType` Map，将按类型查询从 O(n) 优化到 O(1)
- 在卡牌增删时维护索引

### 5.2 批次卡牌 ID 列表
- 每个批次维护 `cardIds` 数组
- 删除批次时可快速定位所有相关卡牌

### 5.3 缓存机制
- 聚合数据（自定义字段、变体类型）使用 `cacheValid` 标记
- 仅在数据变化时重新计算

### 5.4 图片路径推断
- 对没有 `imageUrl` 的卡牌自动推断路径
- 内置卡牌使用固定路径：`/builtin-cards/${cardType}/${cardName}.webp`
- 自定义卡牌路径：`/${batchName}/${cardType}/${cardName}.webp`
- 特殊处理：种族卡牌使用 `class` 字段作为文件名

## 八、关键特性总结

### 内置卡包 vs 自定义卡包

| 特性 | 内置卡包 | 自定义卡包 |
|------|---------|-----------|
| **批次 ID** | 固定 `SYSTEM_BUILTIN_CARDS` | 动态生成 `batch_timestamp_random` |
| **数据来源** | `builtin-base.json` | 用户导入的 JSON 文件 |
| **加载时机** | 系统初始化时首先加载 | 在内置卡包之后加载 |
| **数据持久化** | 每次从 JSON 重新加载 | 保存在 localStorage |
| **可删除** | ❌ 不可删除 | ✅ 可删除 |
| **可禁用** | ✅ 可禁用 | ✅ 可禁用 |
| **禁用状态持久化** | ✅ 保存在索引中 | ✅ 保存在索引中 |
| **卡牌来源标记** | `CardSource.BUILTIN` | `CardSource.CUSTOM` |
| **系统批次标记** | `isSystemBatch: true` | `isSystemBatch: false` |
| **图片路径** | `/builtin-cards/...` | `/${batchName}/...` |

### 批次启用/禁用机制

1. **状态切换**: 所有批次（包括内置）都可以切换启用/禁用状态
2. **影响范围**:
   - 禁用批次的卡牌不会出现在查询结果中
   - 禁用批次的自定义字段不参与聚合
   - 禁用批次在 UI 中显示特殊标记
3. **持久化**: 禁用状态保存在索引文件中，下次加载时恢复

### 数据加载优先级

1. **内置卡包优先**: 确保内置卡牌始终在列表前面
2. **跳过重复加载**: 自定义卡牌加载时跳过已加载的内置批次
3. **禁用状态恢复**: 加载时恢复之前保存的禁用状态

## 总结

卡牌包系统通过统一的 Zustand store 管理所有卡牌数据，使用 localStorage 进行持久化存储。系统明确区分内置和自定义卡牌包，内置卡包具有特殊的保护机制（不可删除）但仍支持禁用功能。通过批次管理、启用/禁用机制、类型索引、缓存等策略，系统在保证数据完整性的同时提供了良好的性能和灵活性。
